# ARIA v3.5 空間推估模組 (Kriging) 開發與除錯紀錄

## 📌 專案概述
本階段目標為完善 ARIA v3.5 災害儀表板中的「空間降雨推估模組」。透過 Ordinary Kriging 演算法，將點位雨量站數據轉換為連續的網格空間資料（GeoTIFF），並產製對應的不確定性地圖（Sigma Map），以輔助防災指揮決策。本次測試針對兩個截然不同的天氣事件進行調校：
* **Event 1**: 局部極端降雨（鳳凰颱風）
* **Event 2**: 廣泛連續性降雨（10/30 共伴豪雨）

---

## 🚀 工作流程 (Workflow)

1. **資料前處理與探索**：將測站雨量資料（z）進行 Log 轉換（`np.log1p`）以處理極端值造成的偏態分佈。
2. **物理參數界定**：捨棄套件的全自動擬合，改為透過觀測數據的變異數（Variance）手動鎖定初始的 Sill（基台值）與 Nugget（塊狀效應）。
3. **網格搜索最佳化 (Grid Search)**：給定符合氣象與地理特性的合理 Range（影響範圍）陣列，讓程式自動疊代 Spherical 與 Exponential 模型，並計算殘差平方和（SSE）。
4. **模型對決與選定**：系統自動比較各參數組合的 SSE，挑選出誤差最小的最佳 Kriging 模型。
5. **不確定性評估**：產製推估雨量圖與 Sigma Map（Variance），比較不同降雨事件的空間連貫性與推估信心。
6. **系統化輸出**：統一將最佳模型推估的網格矩陣翻轉（`np.flipud`）並賦予座標系統，輸出為 GeoTIFF 檔案至 `ARIA_outputs` 資料夾。

## ⚠️ 遭遇問題與解決方案 (Issues & Solutions)

### 問題一：自動擬合失真 (極端值破壞空間結構)
* **狀況**：在使用 `pykrige` 預設的自動擬合時，Event 1（鳳凰颱風）因為蘇澳測站的極端降雨，導致模型被誤導，算出物理上極度不合理、高達十幾萬的 Sill 值與過大的 Range。
* **解決方案**：導入**「物理限制條件」**。直接計算當下觀測數據的變異數作為 Sill 上限（`np.var(z_log)`），並取其 10% 作為 Nugget。強制設定物理天花板，防止模型因為單一極端值而無限發散。

### 問題二：PyKrige 底層參數對位錯誤 (Bug 排除)
* **狀況**：即使給定了正確的初始參數，在執行網格搜索時，算出的最佳 Range（50km）與預期（25km）產生嚴重偏差，且 SSE 數值異常。
* **解決方案**：發現是 `pykrige` 在接收 List 格式（`[sill, range, nugget]`）時可能產生順序錯位。我們改採**「嚴格字典格式 (Dictionary)」**（`{'sill': sill_init, 'range': r_m, 'nugget': nugget_init}`）強制變數對位，成功讓演算法回歸正軌，精準抓出 25km 的最佳解。

### 問題三：Matplotlib 字體環境衝突與洗版警告
* **狀況**：Colab/Jupyter 環境缺乏內建的中文字體（如微軟正黑體），導致繪圖時不斷噴出 `UserWarning` 警告，且圖表顯示亂碼（豆腐塊）。
* **解決方案**：
    1. 實作「純英文繪圖環境」，將圖表的標題與圖例全面改為英文（如 `Best Fit: Spherical with 25km`）。
    2. 使用 `warnings.filterwarnings` 強制將字體警告靜音。
    3. 重置 `plt.rcParams['font.family']` 為預設的 `sans-serif`，徹底清除殘留的字體設定。

## 📊 理論與決策應用探討 (Insights)

1. **Sigma Map 的解讀價值**：
   * **事件差異**：廣泛型豪雨（Event 2）的整體變異數（Sill）較低，測站間距內的預測信心較高；局部極端颱風（Event 1）變異極大，盲區的不確定性劇增。
   * **決策應用**：指揮官可依據 Sigma Map 識別「高 Variance 區域」（即模型瞎猜區）。在這些區域應採取**保守原則**（假設更嚴重的雨量）並**主動派遣機動單位**確認實際災情，而非單純依賴儀表板數值。
2. **演算法特性 (Kriging vs. Random Forest)**：
   * Kriging 基於高斯過程，能計算出空間距離上的「變異數」，原生支援不確定性地圖。
   * Random Forest (RF) 等機器學習模型輸出的是絕對的預測均值，無法直接提供基於地理空間分佈的信心水準。這凸顯了 Kriging 在建置防災系統時不可替代的價值。
3. **動態擬合的必要性**：
   不可為圖方便而將所有降雨事件寫死為同一組參數（如 Range=25km）。不同的天氣系統具有不同的空間尺度，強制套用固定參數會導致 Sigma Map 嚴重失真，進而提供指揮官錯誤的安全感（False sense of security）。
